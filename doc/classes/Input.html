Parameters:

  • [options] Object optional

    Configuration object

    • [instanceName] String optional

      name if the the input instance

    • [inputId] String optional

      DOM-id to give the actual input-tag

    • [inputName] String optional

      DOM-name to give the actual input-tag. Can be usefull in forms

    • [value] String optional

      value to set at the beginning

    • [placeholder] String optional

      html5-placholder to use

    • [disabled] Boolean optional

      defines if input can be edited

    • [skin] String optional

      name of the skin to use. Currently 'phone', 'password', 'url', 'email', 'date', 'time', 'color'. Each skin brings it's own default values. For example the password skin has automatically inputType: 'password'

    • [datepickerOptions] Object optional

      config-object to pass to the datepicker component - you can find possible values here http://bootstrap-datepicker.readthedocs.org/en/release/options.html

    • [colorPickerOptions] Object optional

      config-object to pass to the colorpicker component

    • [frontIcon] String optional

      name of icon to display in front

    • [frontText] String optional

      text to display in front

    • [frontHtml] String optional

      html to display in front

    • [backIcon] String optional

      name of icon to display in front

    • [backText] String optional

      text to display in back

    • [backHtml] String optional

      html to display in back

    • [renderMethod] String optional

      name of a special render method to execute. Currently 'colorpicker', 'datepicker', 'time'. For example 'colorpicker' initializes a colorpicker and sets a css-class

    • [inputType] String optional

      the actual type of the input. e.g. 'text' or 'password'
